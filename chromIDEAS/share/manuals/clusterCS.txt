
Description:
    clusterCS computes chromatin state similarity between samples using multiple metrics: ARI (Adjusted Rand Index), 
RI (Rand Index), NMI (Normalized Mutual Information) and MI (Mutual Information). Additionally, it generates heatmaps 
visualization of chromatin state transition patterns across samples. 
http://jmlr.org/papers/v11/vinh10a.html
Usage:   clusterCS [options] -f <file> -a <cell1> -b <cell2>
Version: 1.0

Required arguments:
  Option                    Description
-----------------------------------------------------------------------------------------------------------------------
  -f <file>                 chromatin state segmentation file (space delimited) generated by chromIDEAS or ideasCS 
                            [Default: None]. Example:
                                #ID CHR POSst POSed cell1 cell2
                                1 chr1 792600 792800 1 1
                                2 chr1 792800 793000 0 0
                                3 chr1 793000 793200 0 0
                                4 chr1 793200 793400 0 0
  -e <emission>             chromatin state emission table file.
  -g <gtf>                  给定GTF文件，也可以是不同区域的bed文件，推荐使用gtf指定，如果你确认在给定区域中CS的分布具有特定意义，则可以考虑使用自定义区域. [Default: None]
  -b <cell2>                Sample name for cell2, it must match those in <file>. [Default: None]

Optional arguments:
  Option                    Description
-----------------------------------------------------------------------------------------------------------------------
  -S <sampleID>             Specify the sample ID(s) to perform the chromatin state clustering. Possible choices: "Merged" and specific sample IDs specified in <input>.
                            "Merged" will perform the chromatin state clustering 同时联合所有样本信息 in <input>. 而当你指定具体的样本ID时，则表示你希望单独对指定的细胞进行独立聚类分析。
                            If multiple sample IDs need to be specified simultaneously, separate them with commas, e.g., 
                            "sample1,sample2,sample3" 这意味着你希望分别对sample1,sample2,sample3这3个样本进行独立的chromatin state clustering分析. [Default: Merged]
  -b <body_num>             [only for Body mode] Specify the number of bins for dividing the gene body region. [Default: 10]
  -u <up_num>               Number of bins covering the upstream region of the transcription start site. [Default: 5]
  -d <down_num>             Number of bins covering the downstream region of the transcription end site. [Default: 5]
  -l <length_level>         Number of transcript length levels. [Default: 10]
  -n <n_HITs>               Number of highly informative transcripts or regions. 这类似单细胞RNAseq分析中的highly variable genes. [Default: 2000]
  -m <HITs_method>          The method to calculate the highly informative transcripts.
  -L <length_cutoff>        The minimal length in bins for transcripts or regions.
  -r <resolutions>          The clustering resolutions. You can set multiple value separated by comma，同时，如果需要指定一个sequences的数值，可以依据start-end-step. 
                            e.g. "0.8,0.9,1.3-1.8-0.1,2"，这意味着同时测试0.8,0.9,1.3,1.4,1.5,1.6,1.7,1.8,2这9个分辨率结果
  -p <nthreads>             Number of parallel processes. [Default: 4]
  -h                        Show this help message and exit.
  -v                        Show program's version number and exit.

For more information visit: https://github.com/YenLab
